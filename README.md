## 搬瓦工和 Cloudflare，各自是什么角色

先简单梳理一下分工，免得后面操作的时候脑子乱。

**搬瓦工（BandwagonHost）**：是你的服务器提供商，负责运行网站程序、存储数据。它的核心优势是线路——CN2 GIA、软银、香港直连这些高端路由，保证了源站到各地的底层传输质量。2012 年成立以来，一直是国内外用户选择最多的 VPS 品牌之一。

**Cloudflare**：是站在你服务器前面的"中间层"，负责 DNS 解析、CDN 分发、DDoS 防御、SSL 证书等一系列上层服务。免费套餐就能用 CDN 和 DDoS 防护，这也是它在建站圈子里被叫做"赛博活佛"的原因。

两者搭配起来，逻辑大概是：用户访问域名 → Cloudflare 节点接收请求 → 转发到搬瓦工源站 → 返回内容。静态资源直接由 Cloudflare 缓存节点响应，不用每次都跑到源站，速度快、流量省。

## 为什么要在搬瓦工 VPS 前面套 Cloudflare

有几个场景，套 CF 几乎是必选项：

**场景一：网站被攻击**。搬瓦工本身没有内置 DDoS 防御能力，如果网站遇到 CC 攻击或流量洗，直接扛很可能带宽打满或触发商家的流量限制策略。接入 Cloudflare 之后，恶意流量在 CF 边缘节点就被过滤掉了，真正到达源站的都是正常请求。

**场景二：需要免费 SSL 证书**。CF 提供一键 SSL，域名解析过去之后直接就能走 HTTPS，省去了手动申请 Let's Encrypt 并配置续期的麻烦。

**场景三：隐藏服务器真实 IP**。套上 CF 的橙色云朵之后，访客看到的是 Cloudflare 的 IP，你的 VPS 真实 IP 被隐藏起来，有效防止被人直接针对源 IP 打。

**场景四：静态资源加速**。图片、JS、CSS 这类静态资源被 CF 缓存之后，用户从离自己最近的 CF 节点取数据，响应速度比每次都跑到你的 VPS 快很多。

不过，搬瓦工 Cloudflare 的组合也有一个需要注意的点：CF 在国内的节点表现参差不齐，晚高峰偶尔会出现延迟上升或丢包。这个问题的解法就是下面要讲的**优选 IP**。

## 配置流程：搬瓦工 VPS 接入 Cloudflare

### 第一步：注册 Cloudflare 账号并添加域名

打开 Cloudflare 官网注册账号，然后选择"Add a Site"，输入你的域名。系统会自动扫描当前的 DNS 解析记录并导入——这一步要检查一遍，确认扫描结果没有漏掉重要记录。

接下来选择套餐，普通建站用免费套餐就够，里面已经包含了 CDN 和基础 DDoS 防护。

### 第二步：修改域名的 DNS 服务器

Cloudflare 会给你两个 NS（Name Server）地址，比如 `ns1.cloudflare.com` 和 `ns2.cloudflare.com`。你需要去你的域名注册商（比如 Namesilo、Namecheap 等）后台，把域名原来的 NS 改成这两个。

修改之后等待生效，通常半小时内大部分地区就能解析到了，官方说法是 24-48 小时全球同步。

### 第三步：添加 A 记录，指向搬瓦工 VPS 的 IP

在 Cloudflare 的 DNS 管理界面，添加一条 A 记录：

- **名称**：`@`（代表根域名）或者 `www`
- **内容**：你的搬瓦工 VPS IP 地址
- **代理状态**：橙色云朵（开启 CDN）或灰色云朵（仅 DNS 解析）

如果你想用 Cloudflare 的 CDN 和防护功能，就选橙色云朵。如果只是用 Cloudflare 做 DNS，不走 CF 节点，就选灰色。

大多数建站场景推荐橙色，能获得 CDN 加速和 DDoS 防护。如果你的搬瓦工线路本身已经非常好（比如香港 CN2 GIA），开不开 CDN 区别不大，有时候甚至走 CF 节点会多一层转发延迟。

### 第四步：SSL/TLS 设置

在 Cloudflare 后台的 SSL/TLS 选项里，推荐把加密模式设置为 **"完全（严格）"**，前提是你的搬瓦工服务器上也安装了有效的 SSL 证书（比如 Let's Encrypt）。这样从访客到 CF 节点、从 CF 节点到源站，全程都走 HTTPS，最安全。

如果服务器没有安装证书，选择"灵活"模式也可以，访客到 CF 走 HTTPS，CF 到源站走 HTTP，但不是最理想的配置。

## 进阶玩法：Cloudflare 优选 IP，解决国内晚高峰卡顿

这是搬瓦工 Cloudflare 组合里比较重要的进阶操作，专门针对国内访问 CF 节点不稳定的问题。

**原理**：Cloudflare 有几千万个 IP，分布在全球各地，国内访问 CF 时实际连接的具体节点是随机的或由 CF 自动调度的。但现实中，不是每个 CF 节点对中国大陆都友好——有些节点延迟低、带宽好，有些节点丢包严重。

优选 IP 就是通过工具测速，在 CF 的大量 IP 中筛选出对你当前网络环境最优的那批 IP，然后把你的域名解析定向到这些 IP 上，绕过 CF 的随机调度，稳定走最优节点。

**实操步骤**（简要版）：

1. 用开源工具 CloudflareSpeedTest（GitHub 上可以找到）对 CF 的 IP 段进行批量测速，得到一批延迟低、丢包率低的优质 IP。

2. 用支持分线路解析的 DNS 服务（比如 DNSPod），对国内用户把你的域名解析到优选出来的 CF IP，而不是让 CF 自动分配。

3. 由于 CF 的优选 IP 会随时间变化，建议配合脚本定期重新测速并更新解析记录，做到动态优选。

这套方案配置完成后，国内用户访问你网站走的就是你精心挑选过的"优质 CF 节点"，晚高峰的卡顿和丢包会明显改善，体验接近直连搬瓦工源站的水平，同时还保留了 CF 的防护层。

## 一些实用建议

**关于是否开启 CF CDN**：如果你的搬瓦工套餐选的是香港或日本软银这类本身延迟就很低的机房，开不开橙色云朵区别其实不大。你可以先灰色云朵跑一段时间测测体验，再决定是否开启 CF CDN。

**关于 CF 免费版的限制**：CF 免费套餐对带宽没有硬性限制，但部分高级功能（如更细粒度的 WAF 规则、更多 Worker 配额）需要付费套餐。一般个人站或中小型商业站，免费套餐完全够用。
